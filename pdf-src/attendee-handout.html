Your first useful AI workflow

A take-home guide for finding recurring legal work, designing a safe workflow, and testing it without turning your firm into a science fair.

June 25–26, 2026 · Orlando, FL

The 5-part workflow test

1. Trigger

When should the workflow start? Example: new consultation scheduled, client uploads documents, weekly metrics updated.

2. Inputs

What facts or files does it need? If the answer is "everything in our firm," narrow it.

3. Output

What should it produce? Checklist, draft email, summary, dashboard, table, or set of next actions.

4. Reviewer

Who checks the output before anyone relies on it?

5. First test

What fake or impossible example can you use to see how it fails?

30-minute Monday test

  1. Pick one workflow from the summit.
  2. Gather three real examples and one fake/impossible example.
  3. Write the job description before opening the tool.
  4. Run the workflow on the fake example first.
  5. Run it on one real example, but do not send/use the output yet.
  6. Mark every correction a human had to make.
  7. Decide: keep testing, narrow scope, or abandon.

Prompts

Workflow finder

You are helping a small law firm identify practical AI workflows. Ask me up to 8 questions about recurring work in my firm. Then return: 1. The top 5 automation candidates. 2. For each: trigger, inputs, output, reviewer, risk level, and first test. 3. One recommendation for the workflow I should build first. Prefer boring, high-frequency work over impressive demos.

Digital employee job description

Create a job description for an AI workflow that handles this recurring task: [TASK] Include mission, inputs, tools it may use, tools it may not use, output format, quality checklist, escalation rules, human review step, and test cases.

PI demand package builder

You are helping draft a personal-injury demand package from source materials. Use only the facts provided. Do not invent treatment, diagnoses, causation, bills, policy limits, or permanency. Inputs: liability summary, medical chronology, medical bills/specials, client impact notes, photos/exhibits, and firm demand-writing style rules. Return: missing information checklist, chronology issues needing human review, demand outline, draft demand letter, exhibit checklist, and risk notes.

Pipeline phase auditor

You are auditing active matters by pipeline phase. Use phase rules and matter data. Do not change records or invent facts. For each matter, identify current phase, required milestones, completed milestones, missing/stale items, blockers, deadline/calendar risk, recommended next action, and owner. Output only: matters needing attention, weird items, clear next actions, and questions a human must answer.

Red flags / green flags

Red flags

  • No reviewer
  • Output goes directly to client
  • Old files with unaudited permissions
  • No source material
  • No owner

Green flags

  • Task happens weekly
  • Output is a draft/checklist/summary
  • Clear reviewer
  • Can test with fake data
  • Success is measurable
