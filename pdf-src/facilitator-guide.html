Cloud AI Summit · Facilitator Guide

Build systems. Not AI party tricks.

A teaching spine for guiding small law firms from "what should I automate?" to workflows, agents, digital employees, and hackathon builds they can use back at the office.

Dan Schnurbusch · Legal Tech Collective · June 25–26, 2026

Core teaching path

1. Reframe AI as staff capacity

Law firms do not need more software. They need reliable work to happen without a person reinventing it every time.

Talk track: "If you would train a human on the task, you can probably train an AI workflow on the first draft of it."

2. Find recurring work

Look for tasks with a trigger, repeatable inputs, predictable output, and clear reviewer.

  • Every consultation gets reviewed.
  • Every probate matter needs document chasing.
  • Every lead needs the same first questions answered.

3. Separate prompt, data, tools, and judgment

A useful AI system is not one mega-prompt. It is a job description plus source material, allowed tools, constraints, and a human approval point.

4. Demo a build, not a miracle

Show the boring parts: files, prompts, failed output, revision, test data, and final artifact. Trust comes from the seams.

5. Make Monday real

Every attendee should leave with one owner, one workflow, one risk boundary, and one 30-minute block to test it.

Teaching lessons

1. Simple law firm website

Brief the site, scaffold with Claude Code/Codex/Hermes/OpenClaw, revise the design, add GA4/Search Console, commit to GitHub, and deploy on Railway.

Notes: files, GitHub, Railway, analytics, mobile check, one ugly first draft getting improved.

2. KPI dashboard

Pick the business question, map the data sources, authenticate safely, build locally with fake data, connect real data, and choose Railway, Tailscale, or local-only access.

Notes: Clio example, read-only auth, env vars, fake data first, narrative summary box.

3. Digital employee

Name the job, define connectors, write the skill, add a scheduled task, start read-only, test bad inputs, and decide how humans supervise it.

Notes: connectors + tools, skills, cron, review gates, managed agents, Claude Cowork, team sophistication.

Hackathon scoring rubric

25 pts

Business value

Solves a real, frequent, painful problem.

20 pts

Workflow clarity

Trigger, inputs, output, owner, and review step are obvious.

20 pts

Usability

A normal staff member could run it without the builder nearby.

15 pts

Risk control

Addresses confidentiality, hallucination, UPL, and permissions.

10 pts

Testing

Includes edge cases, fake contacts, missing fields, or bad inputs.

10 pts

Monday readiness

Credible next step for use or improvement after the summit.

Build ideas

Probate intake checklist generator

Intake notes → missing documents, next steps, client questions.

Client update email drafter

Case notes → plain-English update with next actions.

Consultation prep brief

Intake form → lawyer prep sheet and missing facts.

Medical record collection agent

Provider list/request log → records matrix, missing bills/auths, and next follow-ups.

Medical chronology writer

Medical records/bills → dated treatment timeline, record-cited injuries, gaps, and damages facts.

Demand letter writer agent

Liability summary + medical chronology + bills + firm demand-writing rules → structured demand draft and exhibit checklist.

Document organizer

File inventory → categories, missing items, request list.

Weekly KPI narrator

Metrics → owner briefing: what changed and what to do.

Pipeline phase auditor agent

Case export + phase rules → blocked matters, missing steps, weird items, and owner-specific next actions.

Consultation fit screener

Calendar + intake + email thread → call brief, risk flags, and first questions.

Court hearing prep auditor

Court calendar + matter files → hearing readiness report and missing items.

Filing deadline watch agent

Matter dates + filing rules → deadline watchlist and escalation notes.

Inbox triage and delegation agent

Email stream → priority digest, draft replies, and owner-specific next actions.

Marketing performance analyst

Ads + analytics + lead data → weekly performance brief and recommendations.

Technical site-audit fixer

SEO crawl issues + repo → prioritized fixes and verification notes.

Backlink authority pipeline

Prospects → quality scoring, visual vetting, outreach drafts, and monitoring.

CLE compliance planner

CLE rules + course catalog + calendar → course plan and reminder cadence.

Local file cleanup agent

File inventory + naming rules → safe organization plan and weird files.

Fast judging questions

  1. What job does this replace, support, or accelerate?
  2. What information does it need?
  3. What output does it produce?
  4. Where can it be wrong?
  5. Who reviews it?
  6. What makes this safe enough to try next week?

Prompts and example skills

Connector starter

Ask for read-only auth, minimum scopes, env vars, one safe test fetch, and errors for missing/expired auth.

Dashboard starter

Ask for fake data first, visible metric definitions, stale-data warnings, narrative summary, and Railway/Tailscale/local options.

Skill starter

Ask for a SKILL.md with triggers, inputs, workflow steps, output format, safety rules, mistakes, and verification.

Example skills

Use production-style examples: estate data review, PI records and chronology audit, municipal hearing prep, and marketing/intake brief. Show the mess: connectors, county quirks, missing manifests, source gaps, and human approval lines.
