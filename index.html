Facilitator path

The walkthrough

Use this as your spine. It works as a talk track, a workshop script, and a way to reset the room when demos get noisy.

01

Reframe AI as staff capacity

Law firms do not need more software. They need reliable work to happen without a person reinventing it every time.

Say: “If you would train a human on the task, you can probably train an AI workflow on the first draft of it.”
02

Find the recurring work

Look for tasks with a trigger, repeatable inputs, a predictable output, and a clear reviewer.

  • Every new consultation gets reviewed.
  • Every probate matter needs document chasing.
  • Every lead needs the same first questions answered.
Use estate/probate examples first. Half the room will immediately understand the pain.
03

Separate prompt, data, tools, and judgment

A useful AI system is not one mega-prompt. It is a small job description plus source material, allowed tools, constraints, and a human approval point.

Trigger→Context→Agent→Draft→Human review
04

Demo a build, not a miracle

Show the boring parts: files, prompts, failed output, revision, test data, and the final artifact. That is where trust comes from.

Do not hide the scaffolding. Lawyers trust the method more when they see the seams.
05

Make Monday real

Every attendee should leave with one owner, one workflow, one risk boundary, and one 30-minute block to test it back at the office.

Workshop modules

Three teachable chunks

Each module stands alone. Together they move from approachable to advanced without leaving beginners behind.

Day 120–30 min

Find work worth automating

Teach people how to identify high-value recurring work before they touch a tool.

Use cases

  • Consultation prep from intake notes
  • Client-update email drafts
  • Document request checklists
  • Website FAQ drafts by practice area
Prompt the room: “What task do you explain to staff the same way every month?”
Day 260–75 min

Build a digital employee

Show a working agent-style workflow with a role, inputs, tools, review point, and test cases.

Recommended demo

Build a lightweight firm website or KPI dashboard from structured notes and sample data. It is visible, concrete, and less likely to die in OAuth swamp.

If you do a live desktop-agent demo, narrate every instruction you give it. They need the pattern, not magic.
Hackathon2 hours

Build, present, judge

Help teams choose a narrow workflow, build a rough artifact, and present it with business value and risk boundaries.

Presentation format

  1. Problem in one sentence
  2. Who uses it
  3. What it produces
  4. How a human checks it
  5. What happens Monday

The key distinction

Chatbot: answer a question. Workflow: produce a known output from known inputs. Digital employee: repeat a workflow with tools, memory, boundaries, and review.

Hackathon kit

Build ideas

Pick narrow. Build ugly. Prove value. Polish later.

Estate / Probate

Probate intake checklist generator

Turn intake notes into missing documents, likely next steps, and client questions.

  • Inputs: intake form, death certificate notes, asset list
  • Output: checklist + follow-up email draft
  • Human check: attorney reviews legal path
Estate / Probate

Client update email drafter

Summarize matter status into a plain-English update with next actions.

  • Inputs: case notes, task list, recent filings
  • Output: short email draft
  • Human check: accuracy and tone
Family

Consultation prep brief

Create a pre-call brief from intake answers and flag missing facts.

  • Inputs: intake form
  • Output: lawyer prep sheet
  • Human check: conflict/risk screen
PI

Medical record collection agent

Track providers, request status, missing bills, authorizations, and follow-up dates so the record chase stops living in someone’s head.

  • Inputs: intake notes, provider list, treatment dates, request log
  • Output: records matrix + next follow-ups
  • Human check: paralegal confirms provider/status accuracy
PI

Medical chronology writer

Convert medical records into a dated treatment timeline with injuries, complaints, diagnoses, gaps, and useful demand-letter facts.

  • Inputs: medical records, bills, incident date, client facts
  • Output: chronology table + narrative summary
  • Human check: attorney/paralegal checks citations to records
PI

Demand letter writer agent

Draft a demand package from liability facts, medical chronology, bills, damages notes, and the firm’s demand-writing style guide.

  • Inputs: liability summary, chronology, specials, photos, demand-writing rules
  • Output: structured demand draft + exhibit checklist
  • Human check: attorney reviews liability, causation, damages, and tone
Immigration

Client document organizer

Convert a document dump into categories, missing items, and a client-facing request list.

  • Inputs: filenames or folder inventory
  • Output: document map
  • Human check: legal sufficiency
Business / Employment

SOP bot for recurring advice

Turn internal playbooks into a staff-facing assistant for routine process questions.

  • Inputs: SOP docs
  • Output: answer + source citation
  • Human check: no legal advice to clients
Marketing

Practice-area landing page builder

Generate a draft page from a target client, service, FAQs, and local market.

  • Inputs: service notes, examples, audience
  • Output: page draft or static site
  • Human check: legal claims and ethics
Operations

Weekly KPI narrator

Turn spreadsheet metrics into a short owner briefing: what changed, why it matters, what to do.

  • Inputs: CSV or sheet export
  • Output: dashboard + narrative
  • Human check: data definitions
All Practice Areas

Pipeline phase auditor agent

Audit matters by phase, find blockers, compare required milestones to actual status, and generate a concise “weird items + next actions” report.

  • Inputs: case-management export, phase rules, tasks, calendar/deadline data
  • Output: blocked matters, missing steps, owner-specific next actions
  • Human check: team lead confirms recommendations before changes are made
Intake

Consultation fit screener

Review upcoming consultations before the call and flag bad-fit facts, missing intake answers, likely service path, and questions the lawyer should ask first.

  • Inputs: calendar events, intake form, email thread, fit/disqualifier rules
  • Output: call brief + risk flags + first questions
  • Human check: attorney decides whether/how to proceed
Litigation / Probate

Court hearing prep auditor

Look ahead at hearing dates and verify that filings, exhibits, notices, proposed orders, and client/staff prep tasks are ready.

  • Inputs: court calendar, matter records, filing checklist, local files
  • Output: hearing readiness report + missing items
  • Human check: attorney/paralegal confirms court-specific requirements
Compliance

Filing deadline watch agent

Monitor matter dates against filing rules and surface upcoming, stale, or risky deadlines before they become emergencies.

  • Inputs: matter dates, court deadlines, phase rules, calendar
  • Output: deadline watchlist + escalation notes
  • Human check: attorney confirms legal calculation
Communications

Inbox triage and delegation agent

Sort new email into urgent, delegate, waiting, FYI, and draft-reply buckets so the lawyer sees decisions instead of raw inbox sludge.

  • Inputs: email metadata/body, sender rules, matter map, delegation rules
  • Output: priority digest + draft replies + assigned next actions
  • Human check: lawyer approves sends and sensitive delegation
Marketing

Marketing performance analyst

Combine ads, analytics, search, and lead data into a weekly briefing that explains what changed and what deserves action.

  • Inputs: GA4, Search Console, ad spend, lead quality notes
  • Output: plain-English performance brief + recommendations
  • Human check: owner reviews before budget/content changes
Marketing / SEO

Technical site-audit fixer

Convert crawl issues into prioritized fixes, create small code changes, and produce a before/after verification report.

  • Inputs: Ahrefs/site audit export, repo, deployment rules
  • Output: fix queue, patch/PR, verification notes
  • Human check: owner approves deploys and legal-content edits
Marketing / SEO

Backlink authority pipeline

Find relevant backlink prospects, score quality, visually pre-vet websites, draft outreach, and monitor won/lost links.

  • Inputs: competitor links, prospect database, quality rules, website screenshots
  • Output: scored prospects + outreach drafts + monitoring log
  • Human check: marketer approves outreach before sending
Compliance

CLE compliance planner

Track education deadlines, recommend course blocks, and nudge the lawyer early enough that compliance is boring.

  • Inputs: CLE rules, course catalog, calendar availability, credits completed
  • Output: schedule plan + reminder cadence
  • Human check: attorney confirms jurisdiction and credit eligibility
Operations

Local file cleanup agent

Audit messy Desktop/Downloads folders, group files by likely matter or purpose, and propose safe moves without deleting anything risky.

  • Inputs: file inventory, naming rules, matter list
  • Output: proposed organization plan + weird files
  • Human check: owner approves moves/deletes
Intake

Reception call coach

Create scripts and follow-up questions based on practice area and lead source.

  • Inputs: call type, practice area, ideal client rules
  • Output: script + disqualifiers
  • Human check: compliance and tone

Judging framework

Hackathon scoring rubric

Score the system, not the flash. A boring workflow that saves three hours a week beats a shiny chatbot nobody will trust.

25 pts

Business value

Does this solve a real, frequent, painful problem in the firm?

20 pts

Workflow clarity

Are the trigger, inputs, output, owner, and review step obvious?

20 pts

Usability

Could a normal staff member run this without the builder standing nearby?

15 pts

Risk control

Does it avoid client-confidentiality, hallucination, unauthorized-practice, and data-permission traps?

10 pts

Evidence of testing

Did the team try edge cases, fake contacts, missing fields, or bad inputs?

10 pts

Monday readiness

Is there a credible next step for using or improving it after the summit?

Fast judging questions

  1. What job does this replace, support, or accelerate?
  2. What information does it need to do that job?
  3. What output does it produce?
  4. Where can it be wrong?
  5. Who reviews it?
  6. What would make this safe enough to try next week?

Copy/paste prompts

Prompts that teach the pattern

These are intentionally structured. They teach attendees to define the job before asking for output.

Workflow finder

You are helping a small law firm identify practical AI workflows.

Ask me up to 8 questions about recurring work in my firm. Then return:
1. The top 5 automation candidates.
2. For each: trigger, inputs, output, reviewer, risk level, and first test.
3. One recommendation for the workflow I should build first.

Prefer boring, high-frequency work over impressive demos.

Digital employee job description

Create a job description for an AI workflow that handles this recurring task:
[TASK]

Include:
- Mission
- Inputs it needs
- Tools it may use
- Tools it may not use
- Output format
- Quality checklist
- Escalation rules
- Human review step
- Test cases, including bad or incomplete inputs

Matter/status email draft

Using only the facts below, draft a plain-English client update email.

Rules:
- Do not invent dates, filings, or legal conclusions.
- If information is missing, list it under “Need to confirm.”
- Keep the tone calm, practical, and brief.
- End with the next action requested from the client, if any.

Facts:
[PASTE FACTS]

PI demand package builder

You are helping draft a personal-injury demand package from source materials.

Use only the facts provided. Do not invent treatment, diagnoses, causation, bills, policy limits, or permanency.

Inputs:
- Liability summary
- Medical chronology
- Medical bills/specials
- Client impact notes
- Photos/exhibits
- Firm demand-writing style rules

Return:
1. Missing information checklist.
2. Medical chronology issues that need human review.
3. Demand letter outline.
4. Draft demand letter.
5. Exhibit checklist.
6. Risk notes: weak causation, treatment gaps, pre-existing conditions, unsupported damages.

Write persuasively, but keep every factual claim tied to the supplied materials.

Pipeline phase auditor

You are auditing active matters by pipeline phase.

Use the phase rules and matter data below. Do not change records or invent facts.

For each matter, identify:
1. Current phase.
2. Required milestones for that phase.
3. Completed milestones.
4. Missing or stale items.
5. Blockers, including missing client/staff/court/third-party actions.
6. Deadline or calendar risk.
7. Recommended next action and owner.

Output only:
- Matters needing attention.
- Weird items.
- Clear next actions.
- Questions a human must answer.

Phase rules:
[PASTE RULES]

Matter data:
[PASTE EXPORT]

Hackathon presentation builder

Help me prepare a 3-minute hackathon presentation for the workflow we built.

Use this structure:
1. Problem: one sentence.
2. User: who uses this in the firm.
3. Before: how the work happens now.
4. After: what our workflow produces.
5. Demo path: the 3 screens or outputs to show.
6. Risk boundary: what a human must review.
7. Monday step: what we would do next week.

The 30-minute Monday test

  1. Pick one workflow from the summit.
  2. Gather three real examples and one fake/impossible example.
  3. Write the job description before opening the tool.
  4. Run the workflow on the fake example first.
  5. Run it on a real example, but do not send/use the output yet.
  6. Mark every correction a human had to make.
  7. Decide: keep testing, narrow scope, or abandon.

Red flags

  • The workflow needs judgment but has no reviewer.
  • The output goes directly to a client.
  • The tool can access old files nobody has permission-audited.
  • The prompt says “be accurate” but provides no source material.
  • Nobody owns maintaining it.

Green flags

  • The task happens every week.
  • The output is a draft, checklist, summary, or dashboard.
  • There is a clear human reviewer.
  • The firm can test it with non-client or fake data first.
  • Success is measurable in time saved or errors avoided.

Core takeaway

Do not ask, “How do we use AI?” Ask, “What work should happen the same good way every time?” Then build the smallest system that makes that work easier to repeat.
